# Contributing Guide (Team Workflow)

This repo is our shared codebase for the patient portal project. To keep work organized and avoid conflicts, we follow a simple branch + PR workflow.

## 1) Ground rules
- **Do not push directly to `main`.**
- All changes go through a **feature branch** and a **Pull Request (PR)**.
- The project lead (repo owner) merges PRs after a quick review.

> If you accidentally push to `main`, tell the team immediately — it’s fixable.
